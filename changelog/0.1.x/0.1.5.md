# 0.1.5 — 2026-05-30

## Changed

- **`@cyanheads/mcp-ts-core`** ^0.9.13 → ^0.9.16 — picks up `ctx.enrich()` / `ctx.enrich.notice()` enrichment API, framework skill updates, and dependency refreshes.
- **All twelve collection tools** — `totalCount`, `appliedFilters`, and contextual `notice` guidance now surface in a typed `enrichment` block reaching both the `structuredContent` JSON channel and the `content[]` markdown trailer. Empty-result notices include actionable recovery hints (e.g. "broaden filters", "retry in 30 seconds").
- **`structuredContent` field moves** — the following fields moved from `output` to `enrichment` (value and presence preserved in both channels, key location changed):
  - `wsdot_get_border_waits`, `wsdot_get_ferry_alerts`, `wsdot_get_ferry_terminals`, `wsdot_get_mountain_passes`, `wsdot_get_terminal_space`, `wsdot_get_toll_rates`, `wsdot_get_travel_times`, `wsdot_get_vessel_locations`, `wsdot_search_alerts`: `output.totalCount` → `enrichment.totalCount`
  - `wsdot_get_ferry_routes`: `output.totalCount` and `output.tripDate` → `enrichment.totalCount` / `enrichment.tripDate`
  - `wsdot_search_cameras`: `output.totalCount` and `output.note` → `enrichment.totalCount` / `enrichment.notice` (field renamed)
- **Skills synced** — `add-tool`, `api-context`, `api-linter`, `git-wrapup`, `maintenance`, `setup`, and related framework skill refreshes from the 0.9.14–0.9.16 series.
